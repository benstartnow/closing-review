# 盘后复盘完整版 (closing-review)

A 股收盘复盘报告生成。调取 Tushare Pro 真实收盘数据，输出 8 章完整 HTML 报告。

## 触发场景

用户要求生成盘后复盘/收盘总结/每日复盘报告时调用。

常见触发句式：
- "生成今日盘后总结"
- "收盘复盘"
- "出个盘后报告"
- "写个今日A股复盘"

## 所需前提

1. **Tushare API Token** —— 通过环境变量 `TUSHARE_TOKEN` 传入，或用户对话中提供
2. **Python 3 + requests** —— 确保所需 venv 可用
3. **工作区** —— 生成的 HTML 文件放在 `{{WORKSPACE}}` 或指定路径

## 执行流程

### 1. 获取 Tushare Token

- 优先从环境变量 `TUSHARE_TOKEN` 读取
- 若无环境变量，直接向用户索要 token
- 若已在前序对话中获取过，从内存/项目记忆中复用

### 2. 设置交易日期

- 默认取今天（按 `date +%Y%m%d` 或当前会话日期）
- 用户可指定日期，如"生成7月10日的复盘"

### 3. 运行生成脚本

```bash
VENV=/Users/ben/.workbuddy/binaries/python/envs/default
TUSHARE_TOKEN=xxx TRADE_DATE=20260710 $VENV/bin/python <skill_dir>/scripts/build_report.py
```

脚本会:
1. 从 Tushare 抓取全量数据（指数/全市场个股/涨跌停/行业资金流/大盘资金结构）
2. 聚合成结构化 JSON
3. 生成黑金风格 8 章完整 HTML 报告
4. 输出到 `{{WORKSPACE}}/盘后复盘与次日策略_<日期>.html`

### 4. 交付报告

- 调用 `present_files` 工具预览/交付 HTML 报告
- 将四维度对账数据追加写入 `.workbuddy/memory/YYYY-MM-DD.md`

## 报告章节结构

| 章节 | 内容 |
|------|------|
| 一、全天行情概览 | 六大指数 + 市场宽度 + 涨跌停连板梯队 |
| 二、资金流向全景 | 大盘资金结构 + 板块净额 TOP5 |
| 三、板块龙头解析 | 领涨/领跌板块 TOP5 + 轮动逻辑与持续性 |
| 四、策略复盘对账（核心） | 4.1开盘/4.2操作/4.3个股/4.4规避 + 4.5综合评分 |
| 五、次日开盘预判 | 消息面 + 技术点位 + 三情景概率 |
| 六、次日操作策略 | 仓位建议 + 关注方向 + 规避方向 |
| 七、风险提示 | 六类风险（含极端行情止损失效） |
| 八、对账数据归档 | 四维度回写入长期跟踪日志 |

## 样式

- 黑金高级风格（`background: #0a0b0e` + `gold: #c9a86a`）
- A股配色（涨红跌绿）
- 响应式布局支持手机端
- 完整8章页面，适合收盘后社群/公众号发布

## 注意事项

- **Token 隐私**：token 仅传入脚本环境变量，**不硬编码在脚本中**
- **数据源标注**：报告末尾标注数据来源为 Tushare Pro
- **免责声明**：报告含"不构成投资建议"声明
- **预测性内容**：如用户要求"仅客观分析、无预测"，则跳过第五/六章，或使用 `gen_obj.py` 客观增强版模板
- **个股对账**：第四章个股仅为已发生事实的回顾性对账，标注"非推荐"
